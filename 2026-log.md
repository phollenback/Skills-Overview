# 2026 work log — AZ-104 prep and Azure hands-on

*Last updated: April 1, 2026*

Rough timeline of study and lab work toward the **Microsoft Azure Administrator (AZ-104)** exam. Dates are approximate; tasks are grouped by period and topic.

---

## January 2026 — Foundations and identity

- Mapped **subscriptions, resource groups, and naming** against how real apps are organized (cost boundaries, environments, RBAC scope).
- Studied **Microsoft Entra ID** concepts used in apps: app registration, sign-in flows, and how a SPA talks to a protected API (reinforced by work on **[Movie-Man](https://github.com/phollenback/Movie-Man)** — React client with Entra ID–style auth for cloud PoCs).
- Practiced **role assignments** at subscription vs. resource-group vs. resource scope; compared **built-in roles** to when a custom role might be needed.
- Used **Azure CLI** basics: `az login`, `az account set`, listing groups and resources (repeatable muscle memory for the exam).

---

## February 2026 — Compute, containers, and web hosting

- Packaged and ran **containerized workloads** locally (multi-stage Docker builds, compose) to mirror how **Azure App Service (containers)** and **Azure Container Apps** consume images.
- **Todo-Blazor** (Blazor Server, .NET 8): built a small **.NET + Linux container** workflow — relevant to **App Service** and **Container Apps** deployment paths on the exam.
- **Grailtopia-Azure** (local fork of the Discogs stack): **React + Node API + MySQL** under Docker Compose — used to reason about **multi-container apps**, health checks, env configuration, and what would change when moving from compose to Azure-managed services.
- Reviewed **App Service** plans, scaling, deployment slots, and **managed identity** where secrets should not live in config.

---

## March 2026 — Networking, data touchpoints, and operations

- Reviewed **VNet** basics: subnets, NSGs, service endpoints vs. private endpoints (high-yield for AZ-104 scenario questions).
- Connected container/web scenarios to **Azure DNS**, **load balancing**, and **ingress** patterns at a conceptual level (Container Apps / App Gateway as needed for labs).
- Tied **MySQL-in-compose** work to how **Azure Database for MySQL (Flexible Server)** would replace the container for production (connectivity, firewall rules, TLS).
- **Monitoring and resilience**: **Log Analytics**, **metrics**, **alerts**, and **backup** topics aligned to exam objectives (Application Insights where apps are web-facing).

---

## Cross-cutting: automation and delivery

- **GitHub Actions** pipeline on a small **Express + Docker** project: build, test, push to **GitHub Container Registry** — practice for “artifact exists, deploy to Azure next” workflows.
- Noted **Azure Container Instances** as a lightweight target for one-off or simple container deploys when studying compute options.
